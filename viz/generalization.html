Verdict: Promising infrastructure, overstated results

1. Does the cross-market model generalize?

Not proven. The global chronological 80/20 split means every market appears in the training set — the model saw earlier events from all 25 markets before being tested on their later events. This is held-out-by-time, not held-out-by-market. A true generalization test would train on markets {1–19} and test on {20–25} with zero leakage. That experiment was never run.

The headline LL comparison (+9.17 vs −5.29) is apples-to-oranges. The neural model's loss includes a log-normal time component (time_nll = −12.12) that can go arbitrarily negative when the model outputs low σ. Classical Hawkes LL measures only the intensity integral. Stripping the time term, the neural wallet+bucket loss is ~+2.94 — still better than classical, but a ~8-point gap, not ~14.

2. Do wallet embeddings recover interpretable structure?

Partially. The 500-wallet, 64-dim embedding space does capture activity-level structure: high-frequency traders cluster together in t-SNE. But there is no clear separation of sharp vs reactive vs market-maker archetypes. Profitability coloring (from whale backtest PnL) shows no spatial correlation with embedding position — profitable wallets are scattered uniformly.

The vocabulary covers only the top 500 wallets (≥5 events each). Every other wallet maps to UNK. With ~400K events across potentially thousands of distinct wallets, this is a coarse compression that filters out most participants.

3. Does the model see influence that per-market Hawkes misses?

Unconfirmed. The attention analysis shows strong diagonal dominance — the model primarily attends to immediately preceding events regardless of wallet identity. This is expected from causal masking + recency bias, not evidence of learned cross-wallet influence. The top attention pairs are dominated by temporal proximity, not "wallet A traded → wallet B followed." Cross-city attention exists but is not distinguishable from random baseline given the interleaved data.

4. Recommendation: best city/date for convergence prediction

If continuing: Seoul has the densest coverage (7 markets, March 23–29) and consistently high branching ratios (0.90–1.05). NYC March 31 (event 313358) showed the highest neural LL (+11.80) and a relatively low classical LL (−5.04), suggesting room for the model to exploit structure classical misses.

But first, run the real generalization test: train on the 19 non-Seoul/NYC markets, test on Seoul+NYC with zero data leakage. If the model still beats classical per-market Hawkes on those held-out cities, the generalization claim stands and Seoul/NYC convergence prediction is worth pursuing.

If results don't support it: Pivot to per-market neural TPP (one model per city-date, same architecture but no cross-market claims). The wallet embedding idea still has value for tracking repeat traders within a single market.

5. What would you change about the architecture?

  • Market-held-out training: Leave-one-city-out cross-validation is the minimum bar for "generalization" claims.
  • Separate the time head: The log-normal time prediction dominates the loss and inflates LL. Evaluate wallet prediction accuracy (top-5 accuracy, not NLL) separately from temporal calibration.
  • Expand wallet vocabulary: 500 is too coarse. Use a frequency-based hash or dynamic vocabulary that covers wallets seen ≥2 times, not ≥5.
  • Causal influence validation: Add Granger-causality or do-calculus tests — attention weights alone do not prove influence. Intervene: shuffle wallet A's events and measure prediction degradation for wallet B.
  • Bucket position encoding: Relative position (0=floor, 1=ceiling) already implemented, but bucket MSE=0.06 is nearly trivial. Consider predicting the bucket as a classification (11-way) to better capture discrete structure.
